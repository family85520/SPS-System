## 一、项目概述

### 1.1 项目背景

应急涉安部门承担7×24小时连续值守任务，当前排班工作主要依赖人工方式（如Excel表格、纸质记录），存在以下痛点：

- 排班规则灵活多变（班次时长、班次数、约束条件等），人工排班效率低且容易出错
- 每次排班需反复核对人员连续工作天数、班次间隔、最低人数等约束条件，耗时耗力
- 人员临时请假、换班时需逐一沟通协调，流程繁琐
- 排班调整后通知不及时，容易出现信息不对称

### 1.2 项目目标

建设一套**高灵活、可配置**的排班管理系统，实现：

1. **规则自定义**：管理员可根据实际情况灵活配置排班规则（班次数量、时长、约束条件等），无需修改代码
2. **智能排班**：基于约束规则自动生成排班表，支持手动微调与全局校验
3. **调班流程化**：支持指定换班、开放换班，审批流程可按需启用
4. **信息即时达**：排班发布、调班审批等关键节点通过系统内消息实时通知相关人员
5. **多端覆盖**：PC端满足管理员批量操作需求，移动端满足队员随时查看排班的需求

### 1.3 项目范围

| 在范围内 | 不在范围内 |
|---------|-----------|
| 组织架构管理 | 薪资/考勤统计 |

| 班次模板与约束规则配置 | 第三方IM（微信/钉钉）集成 |
| 自动排班与手动微调 | 视频监控/门禁联动 |
| 排班约束校验 | 多单位/租户模式 |
| 调班换班与审批流程 | 系统本身不提供短信/邮件通知 |
| 系统内消息通知 | |
| 排班数据导出与打印 | |
| PC端 + 移动端（H5/小程序） | |

## 三、功能需求详细说明

### 3.1 基础配置模块

#### 3.1.1 组织架构管理

**功能描述：** 管理员可自定义多层级组织结构，用于人员分组和排班管理。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| ORG-01 | 创建组织层级 | 支持多层级（如大队→中队→班组），每级可自定义名称 |
| ORG-02 | 编辑组织信息 | 修改名称、排序、状态等 |
| ORG-03 | 停用/启用组织 | 停用后该组织不参与排班，但历史数据保留 |
| ORG-04 | 删除组织 | 仅无人员关联的末级组织可删除 |
| ORG-05 | 关联值守岗位 | 每个组织可关联多个值守岗位（如值班室、监控室） |
| ORG-06 | 组织架构树展示 | 树形结构可视化展示，支持展开/折叠 |

**业务规则：**
- 组织层级最多支持4级
- 同级组织名称不可重复
- 有人员关联的组织只能停用，不能删除

---

#### 3.1.2 人员管理

**功能描述：** 管理参与排班的所有人员信息。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| STF-01 | 添加人员 | 姓名、工号、联系方式、所属组织、角色 |
| STF-02 | 编辑人员 | 修改人员基本信息 |
| STF-03 | 人员状态管理 | 在岗/请假/外派/停用，状态变更影响排班 |
| STF-04 | 关联可用岗位 | 设定该人员可以在哪些岗位值守 |
| STF-05 | 设置特殊角色标记 | 如"带班领导""机动人员""新入职"等标签 |
| STF-06 | 人员列表查询 | 按组织、角色、状态筛选 |

**业务规则：**
- 工号全系统唯一
- 停用人员不参与自动排班，但管理员可在手动排班时选用（需确认提示）
- 请假/外派状态的人员自动排除出排班

#### 3.1.3 角色与权限管理

**功能描述：** 定义系统角色及对应的操作权限。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| ROL-01 | 预置角色 | 系统预置四个角色（系统管理员、排班管理员、组长、普通队员） |
| ROL-02 | 权限分配 | 为每个角色勾选可访问的功能模块和操作权限 |
| ROL-03 | 用户分配角色 | 一个用户可分配一个或多个角色 |

**业务规则：**
- 系统管理员角色不可删除，至少保留一个
- 每个组织至少有一个组长角色

#### 3.1.4 班次模板管理

**功能描述：** 管理员自定义班次类型，是排班的核心基础数据。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| SFT-01 | 创建班次模板 | 班次名称、起止时间、时长（自动计算）、颜色标识 |
| SFT-02 | 配置值班领导 | 每班值班领导人数范围（最少~最多），候选人员池 |
| SFT-03 | 配置值班人员 | 每班值班人员人数范围（最少~最多） |
| SFT-04 | 配置适用日期 | 全周适用 / 自定义（如仅工作日适用） |
| SFT-05 | 班次模板复制 | 基于现有模板快速创建新模板 |
| SFT-06 | 启用/停用模板 | 停用的模板不参与自动排班 |

**数据字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 班次名称 | 文本 | 是 | 如"早班""夜班" |
| 起始时间 | 时间 | 是 | 如 08:00 |
| 结束时间 | 时间 | 是 | 如 16:00 |
| 班次时长 | 数值 | 自动 | 自动计算，跨夜班特殊处理 |
| 颜色标识 | 颜色 | 是 | 日历视图展示用 |
| 值班领导-最少人数 | 整数 | 是 | 0表示可不配领导 |
| 值班领导-最多人数 | 整数 | 是 | |
| 值班领导-候选人员池 | 人员列表 | 是 | 从标记为领导/带班的人员中选 |
| 值班人员-最少人数 | 整数 | 是 | |
| 值班人员-最多人数 | 整数 | 是 | |
| 适用日期 | 星期 | 是 | 周一至周日，可多选 |

**业务规则：**
- 班次名称在同一组织下不可重复
- 跨夜班（如 22:00-06:00）时长自动识别并计算为8小时
- 班次时长为0或负数时禁止保存

#### 3.1.5 排班约束规则引擎

**功能描述：** 管理员可灵活配置排班时需要遵守的各类约束条件，支持按需启用/禁用。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| CST-01 | 规则列表管理 | 查看、创建、编辑、启用/禁用约束规则 |
| CST-02 | 规则参数配置 | 每类规则的具体参数可灵活设置 |
| CST-03 | 规则优先级 | 多个规则冲突时按优先级处理 |
| CST-04 | 规则适用范围 | 可指定规则适用于全组织或特定班组 |

**预置约束规则类型：**

| 规则类型 | 可配置参数 | 默认值 |
|---------|-----------|--------|
| 连续工作上限 | 最多连续上班天数 | 5天 |
| 连续工作后最少休息 | 休息天数 | 1天 |
| 班次最少间隔 | 两个班次之间最少间隔小时数 | 8小时 |
| 夜班后最少休息 | 夜班结束后最少休息小时数 | 12小时 |
| 每人每天最多上班数 | 每天最多排几个班 | 1个班 |
| 每周最多工作时长 | 每周累计工作小时上限 | 48小时 |
| 节假日模式 | 正常轮转 / 特殊安排 | 正常轮转 |
| 周末差异化 | 是否对周末单独设置排班规则 | 否 |

**业务规则：**
- 未启用的规则不参与自动排班计算和约束校验
- 管理员在手动排班时可忽略规则警告（系统记录操作日志）
- 约束规则修改后，已发布的排班表不受影响，下次排班时生效

#### 3.1.6 特殊角色排班规则

**功能描述：** 针对特定人员设定个性化的排班约束，覆盖通用规则。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| SPR-01 | 为人员添加特殊规则 | 在人员详情页添加，或在约束规则中指定特定人员 |
| SPR-02 | 特殊规则类型 | 不排某班次、不排某岗位、必须搭配某人、仅排某班次等 |
| SPR-03 | 有效期 | 可设定起止日期，到期自动失效 |
| SPR-04 | 特殊规则优先级 | 个人特殊规则优先于通用规则 |

**典型场景示例：**

| 场景 | 配置方式 |
|------|---------|
| "张三每周三不排晚班" | 人员：张三；规则：不排班次=晚班；条件：每周三 |
| "新队员赵六不能独立值班" | 人员：赵六；规则：必须搭配指定老队员；有效期=入职后3个月 |
| "王队长每月第一周只排带班" | 人员：王队长；规则：仅排角色=值班领导；条件：每月第1周 |
| "李四不能排监控室岗位" | 人员：李四；规则：不可排岗位=监控室 |

### 3.2 排班管理模块

#### 3.2.1 排班主流程

**功能描述：** 排班的核心操作流程，遵循"自动排班 → 手动微调 → 约束校验 → 确认发布"的优先级。

**整体流程图：**

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  自动排班  │───▶│  手动微调  │───▶│  约束校验  │───▶│  确认发布  │
│ (第一步)  │    │ (第二步)  │    │ (第三步)  │    │ (第四步)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
    │                │               │               │
    ▼                ▼               ▼               ▼
 一键生成        拖拽调整        逐条检查         推送通知
 冲突标记        实时提示        校验报告         锁定排班
 生成报告        人员统计        豁免处理         记录日志
```

#### 3.2.2 自动排班

**功能描述：** 系统根据已启用的约束规则和班次模板，自动为指定时间段生成排班表。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| AUTO-01 | 选择排班周期 | 选择起止日期 |
| AUTO-02 | 选择适用班次模板 | 勾选本周期使用的班次模板 |
| AUTO-03 | 选择约束规则集 | 可选择使用哪些约束规则（默认使用全部已启用规则） |
| AUTO-04 | 选择排班范围 | 按组织选择，可选全部或部分班组 |
| AUTO-05 | 一键生成 | 触发自动排班算法 |
| AUTO-06 | 生成结果预览 | 在日历视图中展示排班结果 |
| AUTO-07 | 生成报告 | 展示人员工作时长分布、公平性分析、约束冲突情况 |
| AUTO-08 | 冲突标记 | 约束无法完全满足的位置红色高亮标记 |

**自动排班算法核心逻辑：**

```
输入参数：
  ├─ 排班日期范围
  ├─ 班次模板列表（含值班领导/人员人数范围）
  ├─ 可用人员列表（排除请假/停用/外派人员）
  ├─ 已启用的约束规则列表
  ├─ 特殊角色排班规则
  └─ 历史排班数据（用于公平性计算）

处理流程：
  Step 1: 按优先级排序约束规则
  Step 2: 对排班周期内每一天循环处理
    Step 2.1: 获取当天需排的班次列表
    Step 2.2: 对每个班次
      ├─ 先排值班领导：
      │   ├─ 从领导候选池筛选可用人员
      │   ├─ 应用特殊规则过滤
      │   ├─ 按约束规则过滤（连续工作、间隔等）
      │   ├─ 按公平性打分（休息天数多者优先）
      │   └─ 选择最优人员
      └─ 再排值班人员：
          ├─ 筛选可用人员
          ├─ 应用特殊规则过滤
          ├─ 按约束规则过滤
          ├─ 按公平性打分
          └─ 在人数范围内选择最优人员组合
  Step 3: 全局约束校验
  Step 4: 输出排班表 + 冲突报告

公平性打分因素：
  ├─ 近期休息天数（越多越优先排班）
  ├─ 累计工作时长（越少越优先排班）
  ├─ 夜班次数均衡（夜班少的优先排夜班）
  └─ 周末/节假日均衡
```

#### 3.2.3 手动微调

**功能描述：** 在自动生成排班表的基础上，管理员可手动调整个别班次的人员安排。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| MAN-01 | 拖拽排班 | 将人员从一个班次拖拽到另一个班次 |
| MAN-02 | 点击排班 | 点击班次格子，从下拉列表选择人员 |
| MAN-03 | 移除人员 | 将人员从某个班次移除 |
| MAN-04 | 实时约束提示 | 每次操作实时显示是否违反约束规则 |
| MAN-05 | 人员排班详情 | 点击人员可查看排班明细：已排天数、累计时长、距上限等 |
| MAN-06 | 排班复制 | 将某一周的排班整周复制到另一周 |
| MAN-07 | 撤销/重做 | 支持操作撤销和重做 |

**实时提示示例：**
```
将"张三"拖入 6月5日晚班 时：

⚠️ 提示：
  ├─ 张三已连续上班4天，再排1天将触发连续工作上限（5天）
  ├─ 张三当日早班已排，每人每天最多排1个班
  └─ 当前操作不违反任何强制规则

[继续排入]  [取消]
```

#### 3.2.4 约束校验

**功能描述：** 排班调整完成后，管理员可触发全局约束校验，系统逐条检查所有已启用的约束规则。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| CHK-01 | 一键校验 | 触发全局约束校验 |
| CHK-02 | 校验报告 | 逐条展示规则检查结果 |
| CHK-03 | 豁免处理 | 对警告项可标记"已知豁免"并填写原因 |
| CHK-04 | 返回修改 | 校验不通过时可返回手动微调，修改后重新校验 |

**校验报告样式：**
```
┌────────────────────────────────────────────────────────┐
│  排班校验报告          排班周期：2026年6月1日-6月30日     │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ✅ 已通过（8条）                                       │
│  ├─ 每班最少1人在岗 —— 全部通过                          │
│  ├─ 每班含值班领导（最少0人）—— 全部通过                  │
│  ├─ 班次间隔≥8小时 —— 全部通过                           │
│  ├─ 每人每天最多1个班 —— 全部通过                        │
│  ├─ 最多连续上班5天 —— 全部通过                          │
│  ├─ 连续上班后最少休息1天 —— 全部通过                     │
│  ├─ 夜班后最少休息12小时 —— 全部通过                     │
│  └─ 每周最多工作48小时 —— 全部通过                       │
```

#### 3.2.5 排班发布

**功能描述：** 校验通过后，将排班表正式发布，系统自动通知相关人员。

**功能点：**

| 编号 | 功能 | 说明 |
|------|------|------|
| PUB-01 | 确认发布 | 二次确认后发布排班表 |

**业务规则：**
- 发布前必须完成约束校验（至少运行过一次校验，允许有豁免项）
- 发布操作不可由普通队员执行

### 3.3 调班管理模块

#### 3.3.1 调班模式

系统支持两种调班模式：

**模式A：指定换班**
```
场景：张三和李四协商好互换班次

流程：
  张三发起申请 → 选择与李四互换的班次 → 等待李四确认
  → 等待管理员审批（如开启审批） → 系统自动更新排班 → 通知相关人员
```

**模式B：开放换班（替班申请）**
```
场景：张三某天有事无法上班，需要人替班

流程：
  张三发起替班申请 → 选择无法上班的班次 → 申请发布到班组
  → 其他队员查看并认领 → 等待管理员审批（如开启审批）
  → 系统自动更新排班 → 通知相关人员
```
